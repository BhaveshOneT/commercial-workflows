# Response Draft Templates

## How to Use This File

These templates provide **structural patterns**, not copy-paste text. Adapt the tone, length, and specifics to match the actual email thread. Never use these verbatim — the draft must read as if the user personally wrote it.

---

## NEGATIVE Response Patterns

### Pattern: Graceful Close

```
Hi [Name],

Appreciate you letting me know — totally understand.

If anything changes down the road, happy to pick this back up. Wishing you and the team all the best.

[User's first name]
```

### Pattern: Competitor Acknowledgment

```
Hi [Name],

Thanks for the update. [Competitor] is solid — sounds like you're in good hands.

If you ever want a second opinion or things shift, my door's open.

Best,
[User's first name]
```

### Rules for NEGATIVE drafts:
- 3–4 sentences maximum
- No re-pitching, no "but have you considered..."
- No guilt ("sorry to hear that")
- Acknowledge their decision with respect
- One soft door-open line — not desperate, not pushy
- End with warmth, not a CTA

## NEUTRAL Response Patterns

### Pattern: Referral Follow-Through

```
Hi [Referred Name],

[Original contact] suggested I reach out — we've been chatting about [brief context, 5 words max].

Would love to get 15 minutes on your calendar to see if there's a fit. Would any of these work?

- [Time slot 1]
- [Time slot 2]
- [Time slot 3]

[User's first name]
```

### Pattern: Timing Objection — Value Add

```
Hi [Name],

Totally makes sense — [quarter/timeframe] works. I'll circle back then.

In the meantime, thought this might be useful: [one-sentence description of relevant resource/insight — NOT a product pitch].

Talk soon,
[User's first name]
```

### Pattern: Budget Objection — Low Pressure

```
Hi [Name],

Understood — budget timing is what it is.

Happy to send over a quick overview doc you can share internally when the time is right. No pressure either way.

[User's first name]
```

### Pattern: Technical Objection — Direct Answer

```
Hi [Name],

Good question. [Direct 1–2 sentence answer to their technical concern].

[One follow-up question to keep the conversation moving, e.g., "Are you currently using X for this, or evaluating options?"]

[User's first name]
```

### Pattern: Information Request

```
Hi [Name],

[Direct answer to their question — no preamble].

[Optional: one additional relevant detail they didn't ask about but would find useful].

[One question to advance the conversation].

[User's first name]
```

### Rules for NEUTRAL drafts:
- Match the energy of the original email
- Answer what they asked before adding anything
- One CTA only — don't stack requests
- If they gave a timeframe, respect it (don't try to accelerate)
- For referrals: mention who referred you in the first sentence
- No product pitches disguised as "helpful resources"

## POSITIVE Response Patterns

### Pattern: Meeting Booking

```
Hi [Name],

Great to hear — [1 sentence acknowledging their specific interest/question].

Let's find time to [discuss/demo/walk through] this. Any of these work?

- [Day], [Date] at [Time] [TZ]
- [Day], [Date] at [Time] [TZ]
- [Day], [Date] at [Time] [TZ]

Or grab a time here: [INSERT_BOOKING_LINK]

Looking forward to it,
[User's first name]
```

### Pattern: Decision-Maker Loop-In

```
Hi [Name],

Thanks for connecting me with [DM name] — really appreciate it.

[1 sentence summarizing what you discussed with the original contact].

Would love to set up a quick intro call. Any of these work for both of you?

- [Time slot 1]
- [Time slot 2]
- [Time slot 3]

[User's first name]
```

### Pattern: Pricing/Proposal Request

```
Hi [Name],

Absolutely — happy to walk through pricing.

A quick call would be the fastest way to make sure I'm quoting the right setup for you. Do any of these work?

- [Time slot 1]
- [Time slot 2]
- [Time slot 3]

Or: [INSERT_BOOKING_LINK]

[User's first name]
```

### Rules for POSITIVE drafts:
- Under 120 words — brevity signals confidence
- Thank-you is 1 sentence, not a paragraph
- Acknowledge THEIR specific words, not generic interest
- Always suggest 3 time slots OR a booking link (ideally both)
- Time slots use format: `[Day], [Date] at [Time] [TZ]`
- If calendar not available, use: `[PLACEHOLDER — check your calendar]`
- Don't over-explain what the meeting is for — they already expressed interest
- Don't add attachments, links to resources, or "in the meantime" material — just book the meeting

## Universal Anti-Patterns (NEVER use these)

| Anti-Pattern | Why |
|---|---|
| "I hope this email finds you well" | Filler, universally mocked |
| "Just wanted to follow up" | Passive, adds no value |
| "Per my last email" | Passive-aggressive |
| "Circling back on this" | Use only for timing objections, never as an opener |
| "I'd love to pick your brain" | Presumptuous |
| "Quick question" (when it's not quick) | Dishonest framing |
| "No worries if not!" | Undermines your own ask |
| "Synergy", "leverage", "align", "touchbase" | Corporate jargon |
| "As I mentioned" | Sounds like you're scolding them |
| Exclamation marks on every sentence | Reads as desperate |
| Multiple CTAs in one email | Confuses the reader, reduces response rate |
